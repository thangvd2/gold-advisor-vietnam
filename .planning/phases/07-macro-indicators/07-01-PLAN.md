# Plan 07-01: Macro Data Fetchers + FX Trend Calculator

## Objective

Create data fetchers for DXY (US Dollar Index) and global gold trend, plus a USD/VND trend calculator that reuses the existing Vietcombank fetcher data. Store macro data and compute trend signals for the signal engine.

## Context

- `src/ingestion/fetchers/vietcombank.py` already fetches USD/VND rates and stores them as `product_type='usd_vnd'` in `price_history`
- `src/ingestion/fetchers/base.py` has the `DataSource` abstract base class and `retry` decorator
- `src/analysis/gap.py` shows the DuckDB query pattern for time-series data
- `src/analysis/connection.py` provides `get_duckdb_connection(db_path)`

## Tasks

### Task 1: DXY Index Fetcher (TDD)

**type:** auto
**tdd:** true

**behavior:** `DXYFetcher` extends `DataSource`, fetches current DXY value from a free API (Alpha Vantage or Yahoo Finance via yfinance), returns `FetchedPrice` with `product_type='dxy'`, `price_usd` set to the DXY value.

**implementation:** Use yfinance `^DXY` ticker (same pattern as `YFinanceGoldFetcher`). Store as `FetchedPrice(source='yfinance', product_type='dxy', price_usd=DXY_value, currency='USD')`.

**done:** `DXYFetcher.fetch()` returns a list with one `FetchedPrice` for DXY.

### Task 2: USD/VND Trend Calculator (TDD)

**type:** auto
**tdd:** true

**behavior:** `calculate_fx_trend(db_path, lookback_days=30)` queries stored `usd_vnd` prices from `price_history`, computes a simple moving average trend. Returns dict with `current_rate`, `ma_7d`, `ma_30d`, `trend` (up/down/neutral based on current vs MA), and `change_pct`.

**implementation:** DuckDB query on `price_history WHERE product_type='usd_vnd'`. Compute 7d and 30d moving averages. Trend = 'up' if current > MA * 1.01, 'down' if < MA * 0.99, else 'neutral'.

**done:** `calculate_fx_trend()` returns dict with trend info, or None if insufficient data.

### Task 3: Global Gold Trend Calculator (TDD)

**type:** auto
**tdd:** true

**behavior:** `calculate_gold_trend(db_path, lookback_days=30)` queries stored `xau_usd` prices, computes trend direction and momentum. Returns dict with `current_price`, `ma_7d`, `ma_30d`, `trend` (up/down/neutral), `momentum` (rate of change % over lookback).

**implementation:** DuckDB query on `price_history WHERE product_type='xau_usd' AND source='yfinance'`. Same MA pattern as FX trend.

**done:** `calculate_gold_trend()` returns dict with trend info, or None if insufficient data.

### Task 4: Wire Fetchers into Scheduler

**type:** auto

Add `DXYFetcher` to the scheduler alongside existing sources. Macro data fetches less frequently (every 30 minutes) since it changes slowly.

**done:** DXY data is fetched on schedule and stored in `price_history`.

## Verification

1. `DXYFetcher.fetch()` returns valid `FetchedPrice` with DXY data
2. `calculate_fx_trend()` computes correct MA and trend from stored data
3. `calculate_gold_trend()` computes correct MA and trend from stored data
4. All tests pass
