## Key Design Decisions

- Use **uv** as package manager (per STACK.md). Initialize with `uv init --no-readme` if needed, then `uv add` for dependencies.
- Use **SQLAlchemy 2.0** with `Mapped` syntax and `aiosqlite` for async SQLite.
- Use **pydantic-settings** for configuration (env vars + .env file support).
- FastAPI lifespan event initializes the database (create tables) on startup.
- No Alembic yet — use `metadata.create_all()` for initial schema. Add Alembic in a later phase when schema evolution is needed.

## Database Schema (implement in src/storage/models.py)

Two tables — design for time-series storage:

### price_history
| Column | Type | Purpose |
|--------|------|---------|
| id | Integer (PK, autoincrement) | Row ID |
| source | String(50) | Data source name ("yfinance", "kitco", "sjc", etc.) |
| product_type | String(50) | Product ("xau_usd", "sjc_bar", "nhan_tron") |
| buy_price | Float (nullable) | Buy price in source currency |
| sell_price | Float (nullable) | Sell price in source currency |
| price_usd | Float (nullable) | Price in USD (for international) |
| price_vnd | Float (nullable) | Price converted to VND |
| currency | String(10) | Original currency ("USD", "VND") |
| timestamp | DateTime (UTC) | Time of price observation |
| fetched_at | DateTime (UTC) | When our system fetched it |
| validation_status | String(20) | "valid", "stale", "missing", "anomalous" |

### data_quality_alerts
| Column | Type | Purpose |
|--------|------|---------|
| id | Integer (PK, autoincrement) | Row ID |
| check_type | String(50) | Type of check ("freshness", "anomaly", "missing") |
| severity | String(20) | "warning", "critical" |
| source | String(50) | Which data source triggered this |
| message | String(500) | Human-readable alert description |
| detected_at | DateTime (UTC) | When the check detected the issue |

## Configuration (src/config.py)

Settings class with these fields:
- `app_name: str = "gold_advisor"`
- `database_url: str = "sqlite+aiosqlite:///./gold_advisor.db"`
- `log_level: str = "INFO"`
- `fetch_interval_minutes: int = 5` (used by scheduler in Plan 03)
- `freshness_threshold_minutes: int = 15` (DATA-06 staleness check)
- `anomaly_threshold_percent: float = 10.0` (% jump to flag as anomalous)
