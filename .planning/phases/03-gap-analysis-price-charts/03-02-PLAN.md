**Price field selection logic for chart data:**
- `sjc_bar`: Use `sell_price` (dealer asking price in VND). Average across all dealers per time bucket.
- `ring_gold`: Use `sell_price` (same convention). Average across all dealers per time bucket.
- `xau_usd`: Use `price_vnd` (converted VND/lượng). Do NOT use `sell_price` (that's previous close).

    **Task 1**: Price history service.
    - `src/analysis/prices.py`: `get_price_series(db_path, product_type, range)` function.
    - Uses `get_duckdb_connection()` from connection.py (created in 03-01).
    - DuckDB SQL strategy:
      - For sjc_bar/ring_gold: SELECT time_bucket, avg(sell_price) FROM price_history WHERE product_type=... AND timestamp >= cutoff GROUP BY time_bucket ORDER BY time_bucket
      - For xau_usd: SELECT timestamp, price_vnd FROM price_history WHERE product_type='xau_usd' AND price_vnd IS NOT NULL AND timestamp >= cutoff ORDER BY timestamp
    - Time bucket size: 5 minutes for 1D, 15 minutes for 1W, 1 hour for 1M/1Y (adaptive resolution based on range to keep response size manageable)
    - Return list of {x, y} dicts. x = ISO timestamp string, y = price (float or None).

    - Query params: product_type (enum validation), range (enum validation).
