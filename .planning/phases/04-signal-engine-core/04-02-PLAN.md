  <name>Reasoning + Modes + Signal Persistence</name>
  <files>

  <behavior>
    ## Reasoning Generator (reasoning.py)
    - Input: Signal (from composite.py), current_gap dict
    - Output: One-line string explaining the signal
    - BUY example: "Gap narrowed to 2.8% vs 30-day avg 4.5% — favorable conditions observed for buying"
    - SELL example: "Gap widened to 6.2% vs 30-day avg 4.5% with rising spread — conditions less favorable"
    - HOLD example: "Gap at 4.3% near 30-day avg 4.5% — no strong signal detected"
    - MUST include actual data values (gap_pct, MA value) — never vague
    - MUST NOT use prediction language (no "will", "expected to", "likely")
    - MUST use observational language ("observed", "tracks at", "compared to")
    - When gap data is None: "Insufficient data for gap analysis — no signal generated"
    - Saver mode prefix: "For long-term accumulation: " + reasoning
    - Trader mode prefix: "For timing-precision: " + reasoning

    ## Mode Interpreter (modes.py)
    - `get_mode_weights(mode: SignalMode) -> dict[str, float]`:
      - SAVER: {"gap": 0.4, "spread": 0.1, "trend": 0.5} — trend more important for long-term
      - TRADER: {"gap": 0.6, "spread": 0.3, "trend": 0.1} — gap more important for timing
    - `get_mode_thresholds(mode: SignalMode) -> tuple[float, float]`:
      - SAVER: (0.05, -0.05) — relaxed thresholds
      - TRADER: (0.25, -0.25) — strict thresholds

    ## SignalRecord Storage
    - New SQLAlchemy model: SignalRecord
      - id: int (PK)
      - recommendation: str
      - confidence: int
      - gap_vnd: float | None
      - gap_pct: float | None
      - mode: str
      - reasoning: str
      - factor_data: str (JSON serialization of factors list)
      - created_at: datetime
    - Repository functions:
      - save_signal(session, signal: Signal) -> SignalRecord
      - get_latest_signal(session, mode: SignalMode | None = None) -> SignalRecord | None
      - get_signals_since(session, since_dt: datetime, mode: SignalMode | None = None) -> list[SignalRecord]
  </behavior>
