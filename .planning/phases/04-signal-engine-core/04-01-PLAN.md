  <name>Signal Engine Core: Types + Factors + Composite Scorer</name>

  <behavior>
    ## Signal Types (types.py)
    - `Recommendation` enum: BUY, HOLD, SELL
    - `SignalMode` enum: SAVER, TRADER
    - `SignalFactor` dataclass: name (str), direction (float -1.0 to 1.0), weight (float), confidence (float 0-1)
    - `Signal` dataclass: recommendation (Recommendation), confidence (int 0-100), factors (list[SignalFactor]), reasoning (str), mode (SignalMode), timestamp (datetime)

    ## Gap Signal Factor (gap_signal.py)
    - Input: current_gap dict (from calculate_current_gap), historical gaps list
    - Logic: Compare current gap_pct against 7d_ma and 30d_ma of gap_pct
      - gap_pct significantly below 30d_ma → direction toward BUY (positive)
      - gap_pct significantly above 30d_ma → direction toward SELL (negative)
      - gap_pct near 30d_ma → direction near 0 (neutral)
    - Uses "significantly" = gap is more than 20% away from moving average
    - Returns SignalFactor with name="gap", direction in [-1, 1], weight=0.5

    ## Spread Signal Factor (spread_signal.py)
    - Input: current_gap dict (contains dealer data), historical gaps list
    - Logic: Look at spread (sell - buy) for each dealer in current data
      - Wider spreads (sell-buy gap large) → reduce confidence (direction slightly negative)
      - Tighter spreads → more favorable conditions (direction slightly positive)
    - This is a secondary/modulating factor, not primary
    - Returns SignalFactor with name="spread", direction in [-0.5, 0.5], weight=0.2

    ## Trend Signal Factor (trend_signal.py)
    - Input: historical gaps list (with ma_7d, ma_30d), current_gap dict
    - Logic: Compare current SJC sell price vs 7d and 30d moving averages of gap
      - If gap_vnd has been narrowing (trending down) → positive direction (buy conditions improving)
      - If gap_vnd has been widening (trending up) → negative direction (sell conditions)
    - Moving average cross signals: 7d_ma crossing above 30d_ma → bearish; below → bullish
    - Returns SignalFactor with name="trend", direction in [-1, 1], weight=0.3

    ## Composite Scorer (composite.py)
    - Input: list[SignalFactor], mode (SignalMode)
    - Logic: Weighted sum of (direction × weight) for all factors → raw score
      - raw_score > 0.15 → BUY
      - raw_score < -0.15 → SELL
      - otherwise → HOLD
    - Confidence = weighted average of individual factor confidences, scaled to 0-100
    - Returns Signal with recommendation, confidence, factors list, empty reasoning (filled later), mode, timestamp
    - Saver mode: shift thresholds (0.15 → 0.05 for BUY, -0.15 → -0.05 for SELL) — easier to trigger action
    - Trader mode: tighten thresholds (0.15 → 0.25 for BUY, -0.15 → -0.25 for SELL) — requires stronger signal

    ## CRITICAL PITFALLS (from PITFALLS.md):
    - NEVER use LLM for numerical signals — all computation is pure Python
    - Use directional accuracy, not price levels
    - Confidence ranges, not point estimates
    - Never use "predict" language — use "tracks", "analyzes", "observes"
    - These factors compute directional signals from observable data, NOT predictions
