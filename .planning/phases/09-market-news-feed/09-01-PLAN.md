# Plan 09-01: News Fetcher - RSS Feed Reader

## Objective

Build an async RSS feed reader that fetches gold market news from configurable RSS/Atom feeds. Uses httpx to fetch XML feeds and parse them with Python's built-in `xml.etree.ElementTree`. No LLM processing — just headline, link, source, date, and optional excerpt extraction.

## Context

- Uses existing `retry` decorator from `src/ingestion/fetchers/base.py`
- httpx is the project's async HTTP client (already a dependency)
- News sources will be configurable via a list in config
- Target feeds: VNExpress Economy (RSS), Vietnam Investment Review, Kitco News RSS

## Tasks

### Task 1: News data models

**type:** auto
**tdd:** true

Create Pydantic models for parsed news articles and a NewsFetcher class interface.

**behavior:** A NewsArticle model with fields: title (str), url (str), source (str), published_at (datetime|None), excerpt (str|None), category (str|None). A NewsFetcher class that accepts a list of feed URLs and fetches all articles.

**implementation:** Create `src/ingestion/news/models.py` with Pydantic `NewsArticle` and `src/ingestion/news/fetcher.py` with `NewsFetcher` class.

**done when:** `NewsArticle` validates correctly, `NewsFetcher` class exists with `fetch()` method.

---

### Task 2: RSS feed parser

**type:** auto
**tdd:** true

Parse RSS 2.0 and Atom feed XML into NewsArticle objects.

**behavior:** Given raw XML bytes from an RSS/Atom feed, extract all items as NewsArticle objects. Handle both RSS 2.0 `<item>` and Atom `<entry>` elements. Extract: title, link, published date, optional description/summary as excerpt.

**implementation:** Create `src/ingestion/news/parser.py` with `parse_rss_feed(xml_bytes: bytes, source_name: str) -> list[NewsArticle]` function. Detect feed type (RSS vs Atom) and parse accordingly.

**done when:** RSS 2.0 and Atom XML samples are correctly parsed into NewsArticle lists.

---

### Task 3: NewsFetcher with httpx + retry

**type:** auto
**tdd:** true

Wire the fetcher to fetch feeds via httpx with retry decorator.

**behavior:** NewsFetcher.fetch() fetches all configured feeds concurrently, parses each, deduplicates by URL, and returns sorted articles (newest first). Handles network errors gracefully (log + skip failed feeds).

**implementation:** Complete `src/ingestion/news/fetcher.py` with async fetch logic using httpx.AsyncClient and the existing `retry` decorator.

**done when:** NewsFetcher fetches from mock HTTP, returns deduplicated sorted articles, handles errors.
