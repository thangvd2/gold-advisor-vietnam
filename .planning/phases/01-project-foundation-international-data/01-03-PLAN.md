## Key Design Decisions

- **APScheduler 3.11.2** with `BackgroundScheduler` (per STACK.md — APScheduler 4.x is alpha). Run scheduler in a background thread alongside FastAPI's async event loop.
- **Quality checks** (from PITFALLS.md §3): Check after EVERY fetch, not periodically. Each stored record gets a validation_status.
- **Staleness check**: Compare `fetched_at` of latest record per source against `now() - freshness_threshold_minutes`. Flag as "stale" if exceeded.
- **Anomaly check**: Compare latest price against previous price for same (source, product_type). If change > anomaly_threshold_percent, flag as "anomalous".
- **Missing check**: If a fetch returns empty list (source failure), immediately create a "critical" alert.
- **Pipeline flow**: fetch → validate individual record → convert to VND → store with validation_status → run post-store quality checks → update validation_status if needed → create alerts for failures.

       - `check_freshness(session, source, product_type, threshold_minutes) -> DataQualityAlert | None`:
         Query latest PriceRecord for (source, product_type). If `fetched_at < now - threshold_minutes`, create and return a "warning" severity alert. Otherwise return None.
       - `check_anomaly(session, new_record, threshold_percent) -> DataQualityAlert | None`:
         Query previous PriceRecord for same (source, product_type). If price change > threshold_percent (comparing price_usd or price_vnd), create and return a "warning" severity alert. Handle case where no previous record exists (first run — not anomalous).

       - `async def fetch_and_store(session, gold_fetcher, fx_fetcher, settings) -> dict`:
         1. Fetch gold prices: `await gold_fetcher.fetch()`
         2. If empty → save "critical" quality alert for missing data, return {"status": "failed", "alerts": 1}
         3. Fetch FX rate: `await fx_fetcher.fetch()`
         4. For each FetchedPrice:
            a. If FX rate available and price is in USD: compute price_vnd using convert_usd_to_vnd_per_luong
            b. Set validation_status = "valid" initially
            c. Save to DB via repository.save_price()
         5. Run quality checks on each (source, product_type) pair
         6. If any alerts found: update the corresponding PriceRecord's validation_status to the alert severity
         7. Return {"status": "ok", "prices_saved": N, "alerts": M}
       - `async def fetch_and_store_all(sources: list[DataSource], settings) -> dict`:
         Convenience function that creates a session, iterates sources, calls fetch_and_store for each.
