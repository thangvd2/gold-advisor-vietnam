# Plan 09-03: Dashboard News Section + Scheduler Integration

## Objective

Add a news feed section to the dashboard and integrate periodic news fetching into the scheduler.

## Context

- Dashboard uses Jinja2 partials with HTMX auto-refresh
- Follow existing partial pattern from `partials/macro_card.html`
- Scheduler already runs periodic jobs via APScheduler BackgroundScheduler
- News section goes below charts, above footer in dashboard.html

## Tasks

### Task 1: News dashboard partial + API

**type:** auto
**tdd:** true

Create a news feed partial for the dashboard and a data endpoint.

**behavior:** A card showing the latest 10 news items with: headline (linked to source), source name, published date (relative time), optional category badge. State Bank category items get a gold accent badge. Empty state shows "No news available".

**implementation:** Create `templates/partials/news_card.html`. Add `/dashboard/news` JSON endpoint and `/dashboard/partials/news` HTML endpoint to dashboard router. Add news section to `templates/dashboard.html`.

**done when:** Dashboard renders news card, shows articles with headline/source/date, empty state works.

---

### Task 2: Scheduler integration for news fetching

**type:** auto
**tdd:** true

Add periodic news fetch job to the scheduler.

**behavior:** News is fetched every 30 minutes (separate from price fetch interval). Fetched articles are deduplicated against existing DB entries (by URL) and new ones are stored.

**implementation:** Create `src/ingestion/news/store.py` with `fetch_and_store_news()` function. Add scheduler job in `src/ingestion/scheduler.py`. Add `news_fetch_interval_minutes` to config (default 30).

**done when:** Scheduler job runs, fetches from mock, deduplicates, stores new articles.

## Verification

- `pytest tests/test_news_template.py -v` passes
- `pytest tests/test_news_scheduler.py -v` passes
- Dashboard HTML contains news section placeholder with HTMX
