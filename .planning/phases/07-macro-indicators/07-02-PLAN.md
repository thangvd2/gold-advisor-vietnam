# Plan 07-02: Macro Signal Factors + Composite Integration

## Objective

Create macro signal factors (FX trend, DXY strength, gold trend) and integrate them into the existing composite scorer. Update mode weights to include macro factors.

## Context

- `src/engine/types.py` defines `SignalFactor` with `name`, `direction`, `weight`, `confidence`
- `src/engine/gap_signal.py`, `spread_signal.py`, `trend_signal.py` show the factor pattern
- `src/engine/composite.py` sums `direction * weight` for all factors
- `src/engine/modes.py` defines `MODE_WEIGHTS` per mode
- `src/engine/pipeline.py` assembles all factors and calls `compute_composite_signal`
- `src/analysis/macro.py` (created in 07-01) provides `calculate_fx_trend()` and `calculate_gold_trend()`

## Tasks

### Task 1: FX Trend Signal Factor (TDD)

**type:** auto
**tdd:** true

**behavior:** `compute_fx_signal(db_path, weight=0.1)` calls `calculate_fx_trend()`, converts the FX trend into a `SignalFactor`. VND weakening (FX up) is bullish for gold in VND (direction positive for BUY). VND strengthening (FX down) is bearish.

**implementation:** If trend='up' → direction=0.3 (VND weakening → gold more expensive → buy signal), if trend='down' → direction=-0.3, if neutral → direction=0.0. Scale confidence by change_pct magnitude. If no data → confidence=0.0.

**done:** Returns `SignalFactor(name='fx_trend', ...)` with proper direction.

### Task 2: Gold Trend Signal Factor (TDD)

**type:** auto
**tdd:** true

**behavior:** `compute_gold_trend_signal(db_path, weight=0.1)` calls `calculate_gold_trend()`, converts gold trend into a `SignalFactor`. Rising gold → positive direction (momentum). Falling gold → negative direction.

**implementation:** If trend='up' → direction=0.3 (momentum play), if trend='down' → direction=-0.3, neutral → 0.0. Confidence based on momentum magnitude.

**done:** Returns `SignalFactor(name='gold_trend', ...)` with proper direction.

### Task 3: Update Mode Weights and Pipeline (TDD)

**type:** auto
**tdd:** true

**behavior:** Add `fx_trend` and `gold_trend` weights to `MODE_WEIGHTS`. Update `compute_signal()` in pipeline to call the new macro factors and include them in the composite.

**implementation:**
- SAVER: `{"gap": 0.3, "spread": 0.1, "trend": 0.4, "fx_trend": 0.1, "gold_trend": 0.1}`
- TRADER: `{"gap": 0.5, "spread": 0.2, "trend": 0.1, "fx_trend": 0.1, "gold_trend": 0.1}`
- Pipeline adds `fx_factor` and `gold_trend_factor` to the factors list

**done:** Signal includes 5 factors (gap, spread, trend, fx_trend, gold_trend) with updated weights.

### Task 4: Update Reasoning to Include Macro Context

**type:** auto

Append macro context to the reasoning string. E.g., "USD/VND trending up (VND weakening) — gold likely to be more expensive." Keep it brief.

**done:** Reasoning includes macro factor context when macro data is available.

## Verification

3. Composite signal now includes 5 factors with new weights
4. Reasoning includes macro context
