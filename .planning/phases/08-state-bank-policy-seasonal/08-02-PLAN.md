# Plan 08-02: State Bank Policy Model

## Objective

Build a State Bank policy event tracking system with manual admin API for recording policy events (import approvals, gold auctions, interventions). Policy events OVERRIDE all other signals — they are first-class, not just another factor.

## Context

- Per PITFALLS.md: "State Bank actions override all other signals — must be first-class"
- Per PITFALLS.md: "Build a manual override. When a major policy change occurs, the system should default to 'reduced confidence' mode"
- Per CONTEXT.md: "State Bank policy: start with manual override (admin endpoint to flag events), auto-detection deferred"
- `src/storage/models.py` has SQLAlchemy models for price data
- `src/storage/database.py` provides async_session

## Tasks

### Task 1: Policy Event Model (TDD)

**type:** auto
**tdd:** true

**behavior:** `PolicyEvent` SQLAlchemy model with fields:
- `id`: Integer primary key
- `event_type`: String (choices: "import_approval", "auction", "intervention", "regulation_change", "inspection")
- `description`: Text (human-readable description of the event)
- `impact`: String (choices: "bullish", "bearish", "neutral", "uncertain")
- `severity`: String (choices: "high", "medium", "low")
- `effective_date`: DateTime (when the event takes effect)
- `expires_at`: DateTime (when the event stops being relevant, nullable)
- `is_active`: Boolean (whether event is currently in effect)
- `created_at`: DateTime (when the event was recorded)

**implementation:** Add model to `src/storage/models.py`. Create Alembic migration.

**done:** Model exists with all fields, migration created.

### Task 2: Policy Signal Calculator (TDD)

**type:** auto
**tdd:** true

**behavior:** `compute_policy_signal(db_path: str) -> dict` queries active policy events from the database and returns:
- `active_events`: list of active policy events
- `has_override`: bool (True if any high-severity active event exists)
- `override_type`: str or None ("bullish", "bearish", or None)
- `confidence_cap`: float (maximum allowed confidence: 0.3 for high-severity, 0.6 for medium, 1.0 for low/none)
- `summary`: str (human-readable summary of active events)

**implementation:** Query PolicyEvent where is_active=True and (expires_at is NULL or expires_at > now). Aggregate severity into confidence cap. High severity with bearish impact → confidence_cap=0.3, bullish → 0.3 (still cap, but signal more nuanced).

**done:** Returns correct policy override data from active events.

### Task 3: Admin API for Policy Events (TDD)

**type:** auto
**tdd:** true

**behavior:** Admin API endpoints:
- `POST /api/admin/policy-events` — create a new policy event (requires JSON body with event_type, description, impact, severity, effective_date, optional expires_at)
- `GET /api/admin/policy-events` — list all policy events (with optional `active_only=true` query param)
- `DELETE /api/admin/policy-events/{id}` — soft-delete (set is_active=False) a policy event

**implementation:** Create `src/api/routes/admin.py` router. Use async SQLAlchemy session. Validate event_type, impact, severity enums. Return created event or list.

**done:** CRUD operations work for policy events via API.
