# Plan 07-03: Macro Dashboard Section + API Endpoints

## Objective

Add a macro indicators section to the dashboard showing USD/VND rate with trend, DXY index, global gold trend, and real interest rate level. Create API endpoints for macro data and an HTMX-powered dashboard partial.

## Context

- Dashboard uses Tailwind CSS with gold/charcoal dark theme
- Partials use HTMX for live refresh (see `templates/partials/signal_card.html` pattern)
- API routes are in `src/api/routes/dashboard.py`
- Dashboard layout in `templates/dashboard.html`
- `src/analysis/macro.py` provides `calculate_fx_trend()` and `calculate_gold_trend()`
- `src/analysis/connection.py` provides DuckDB access

## Tasks

### Task 1: Macro API Endpoints (TDD)

**type:** auto
**tdd:** true

**behavior:** Add endpoints to dashboard router:
- `GET /dashboard/macro` — returns JSON with `fx_trend`, `gold_trend`, `dxy` (latest DXY value from DB)

**implementation:** Query `price_history` for latest DXY value, call `calculate_fx_trend()` and `calculate_gold_trend()`. Return combined dict.

**done:** API returns macro data with all indicators.

### Task 2: Macro Dashboard Partial (TDD)

**type:** auto
**tdd:** true

**behavior:** `GET /dashboard/partials/macro` returns an HTML partial showing:
- USD/VND rate with up/down arrow and change percentage
- Global gold trend with direction indicator
- DXY value with trend indicator
- Compact card layout matching existing dashboard style

**implementation:** Create `templates/partials/macro_card.html` with Tailwind CSS styling matching the gold/charcoal theme. Use the same card pattern as `signal_card.html`.

**done:** Partial renders with all macro data, graceful empty state when data unavailable.

### Task 3: Wire Macro Section into Dashboard Layout

**type:** auto

Add the macro card to `templates/dashboard.html` between the signal card and the gap/prices grid. HTMX loads it with 30-second refresh.

**done:** Dashboard displays macro section with live data.

## Verification

1. `GET /dashboard/macro` returns valid JSON
2. `GET /dashboard/partials/macro` returns styled HTML
3. Dashboard shows macro section with proper styling
4. Partial handles empty data gracefully
5. All tests pass

## Output

- `src/api/routes/dashboard.py` — Updated with macro endpoints
- `templates/partials/macro_card.html` — Macro dashboard partial
- `templates/dashboard.html` — Updated with macro section
- `tests/test_macro_api.py` — Macro API tests
- `tests/test_macro_template.py` — Macro template tests
